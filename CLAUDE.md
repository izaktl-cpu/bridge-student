# כללי עבודה

1. חסוך טוקונים.
2. אל תסרוק את כל הפרויקט אלא אם אין בררה.
3. אל תשנה קבצים שלא בקשתי.
4. אל תחזיר קבצים מלאים אלא אם בקשתי במפורש.
5. החזר רק diff — קטעי קוד שהשתנו בלבד.
6. אל תסביר באריכות.
7. לפני שינוי, ציין בקצרה איזה קובץ אתה משנה.
8. אם יש ספק – שאל שאלה אחת קצרה בלבד.
9. שמור על העיצוב הקיים.
10. בצע רק את המשימה הנוכחית.
11. לפני כל שינוי ב-engine או ב-lesson — קרא תחילה את memory/bridge_bidding_rules.md ובדוק שהשינוי תואם את הטבלאות שם. אם יש סתירה — שאל לפני שמשנה.
12. אין לבצע שום שינוי בקוד ללא אישור מפורש מהמשתמש.

# כלל תצוגת טקסט מעורב (עברית + אנגלית/מספרים/הכרזות)

כל קטע לטיני — מספר, מילה באנגלית, או הכרזה (כמו `2♣` / `1NT` / `♠♥♦♣`) —
שמופיע בתוך משפט בעברית **חייב** להיות עטוף ב-`<bdi>`.

```html
<!-- ❌ נשבר -->
<p>פתח 1NT עם 15 נקודות</p>

<!-- ✅ תקין -->
<p>פתח <bdi>1NT</bdi> עם <bdi>15</bdi> נקודות</p>
```

חלופה ב-CSS להחלה על הרבה הכרזות בבת אחת:

```css
.bid, .num { unicode-bidi: isolate; }
```

לטקסט דינמי או שדה input שלא ידוע מראש אם הוא עברי או לטיני — הוסף `dir="auto"`.

החל את הכלל בכל מקום שמייצרים בו הכרזה או מספר בתוך טקסט עברי — ללא יוצא מן הכלל.
